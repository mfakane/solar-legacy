Solar

* 概要
普通の見た目に微妙な中身の Twitter クライアントです。
Solar は、取り扱うのに面倒なクライアントです。どうかご理解ください。

** 名前について
作った当時は暑かったんです

* 必要環境
- Windows XP SP3 以上
- .NET Framework 4 Full
  →http://msdn.microsoft.com/ja-jp/netframework/ff687189.aspx

* 拡張設定
設定ファイルである Solar.slconf を直接テキストエディタで開くと通常ではできない設定をいじれるかもしれません。
新しいバージョンで追加された設定は、新しいバージョンを一旦起動して終了してからでないと項目が現れません。注意してください。

- <ConnectionSettings>
  : EnableUserStreams |
    True の場合、User Streams によるリアルタイム更新を使用します。
  : UseHttps |
    True の場合、接続に HTTPS を使おうとします。
  : WebProxyHost |
    プロキシを使用する場合、プロキシのホスト名または IP アドレスを入力します。
  : WebProxyPort |
    プロキシを使用する場合、プロキシのポートを入力します。
  : WebProxyUser |
    プロキシが認証が必要な場合、プロキシのユーザ名を入力します。
  : WebProxyPassword |
    プロキシが認証が必要な場合、プロキシのユーザ名を入力します。

- <InterfaceSettings>
  : CollapsePostBox |
    True の場合、投稿ボックスの非フォーカス時に小さくなります。
  : PostByControlEnter |
    True の場合、Ctrl+Enter で投稿します。
    False の場合、Enter で投稿します。
  : MinimizeToTray |
    True の場合、メインウィンドウ最小化時にタスクトレイに格納します。
  : CloseToTray |
    True の場合、メインウィンドウを閉じようとするときに代わりにタスクトレイに格納します。
  : UseQuickSearch |
    True の場合、タイムライン表示の上にクイック検索ボックスを表示します。
  : UseIncrementalQuickSearch |
    True の場合、クイック検索ボックスでインクリメンタル検索します。
    False の場合、Enter で検索します。
  : UseDeferredScrolling |
    True の場合、スクロールバーを離した瞬間のみスクロールします。
    False の場合、スクロールバーを移動するたびにスクロールします。
  : AccountBoxStyle |
    DropDownRight の場合、コンボボックスを投稿ボックス右側に表示します。
    DropDownLeft の場合、コンボボックスを投稿ボックス左側に表示します。
    CheckBox の場合、チェックボックスを投稿ボックス下端に並べます。CheckBox の場合、複数のアカウントへの同時投稿が可能です。中クリックでそのアカウントだけをチェック可能です。
    なおアカウントボックスはシングルアカウント環境では表示されません。
  : NotifyLocation |
    BottomRight の場合、右下から新着ポップアップを表示します。
    BottomLeft の場合、左下から新着ポップアップを表示します。
    TopRight の場合、右上から新着ポップアップを表示します。
    TopLeft の場合、左上から新着ポップアップを表示します。

- <TimelineSettings>
  : BorderThickness |
    項目の枠の幅を指定します。, 区切りで、要素が一つの場合はすべての辺、二つの場合は左右および上下、四つの場合は左上右下の順で設定します。
  : AutoResetNewCount |
    True の場合、取得のたびに新着数をリセットします。
    False の場合、新着数を手動リセットするまで加算されます。
  : ResetNewCountOnTabShow |
    True の場合、タブを表示したときに新着数をリセットします。
  : ResetNewCountOnTabHide |
    True の場合、タブを切り替えたときに新着数をリセットします。
  : MaxStatuses |
    一覧の最大保持件数です。
    数を増やすとよりスクロール取得や更新時の件数の保持が可能になります。数を減らすと少なくなりますが軽くなります。
  : DoubleClickAction |
    つぶやきをダブルクリックした際の動作を指定します。
    None の場合、なにもしません。
    Copy の場合、クリップボードにコピーします。
    ReplyTo の場合、返信します。
    Quote の場合、引用します。
    Favorite の場合、お気に入り / お気に入り解除します。
    StatusDetails の場合、ブラウザで開きます。
    UserDetails の場合、ユーザウィンドウを開きます。
    NearStatuses の場合、前後 5 件のつぶやきを取得します。
  : ShowIconShadowIfAvailable |
    True の場合、ユーザアイコンやコマンドアイコンに影が利用できる環境の場合、影を表示します。
  : ShowStatusShadow |
    True の場合、項目の下部分に影グラデーションを表示します。
  : NewStatusColor |
    新着のハイライト色です。色名、#RRGGBB、#AARRGGBB の形式が使用できます。
  : AlternateStatusColor|
    偶数項目の背景色です。
  : HighlightStatusRepliedBySelected |
    True の場合、選択中のつぶやきの返信先をハイライトします。
  : HighlightStatusRepliedBySelectedColor |
    上記のハイライト色です。
  : HighlightStatusReplyingToSelected |
    True の場合、選択中のつぶやきへの返信をハイライトします。
  : HighlightStatusReplyingToSelectedColor |
    上記のハイライト色です。
  : HighlightStatusHavingSelectedSender |
    True の場合、選択中のつぶやきと同じ送信者のつぶやきをハイライトします。
  : HighlightStatusHavingSelectedSenderColor |
    上記のハイライト色です。
  : HighlightFavorited |
    True の場合、お気に入りしたつぶやきをハイライトします。
  : HighlightFavoritedColor |
    上記のハイライト色です。

- <PostSettings>
  : AutoShortenUrl |
    同名の URL 短縮スクリプトがある場合、それを利用し投稿時に自動的に短縮されていない URL を短縮します。
  : AutoSwitchAccount |
    True の場合、返信や Retweet 等をするときにそれを取得したアカウントに自動的に投稿先アカウントを切り替えます。

- <KeySettings>
  : CopyStatus |
    コピーのキーバインドです。
  : CopyStatusUri |
    URL をコピーのキーバインドです。
  : CopyStatusUser |
    ユーザ名をコピーのキーバインドです。
  : DirectMessage |
    ダイレクトメッセージを送るのキーバインド
  : DeleteStatus |
    削除のキーバインドです。
  : ReplyToStatus |
    返信のキーバインドです。
  : RetweetStatus |
    Retweet のキーバインドです。
  : QuoteStatus |
    引用のキーバインドです。
  : ReplyToDetails |
    返信履歴のキーバインドです。
  : FavoriteStatus |
    お気に入りのキーバインドです。
  : Refresh |
    更新のキーバインドです。
  : SearchStatuses |
    つぶやきの検索のキーバインドです。
  : SearchUsers |
    ユーザの検索のキーバインドです。
  : SearchCache |
    キャッシュの検索のキーバインドです。
  : QueryCache |
    キャッシュへクエリのキーバインドです。
  : SearchQuick |

* リアルタイム更新について
- Twitter User Streams API を使用し、リアルタイムに発言を取得更新できます。
- 接続中はメインウィンドウ右下の API Rate Limit をマウスオーバーすると出てくるリセット時間の右側に [Streaming] と表示されます。
- リアルタイム更新が使用可能なフィルタソースは以下の通り。
  - ホーム
  - 返信
  - 送信済み
  - RT
  - 受信メッセージ
  - 送信メッセージ
  - お気に入り
  これら以外のソースは従来通り定期取得による更新が使用されます。
- Twitter User Streams API は予告なく一時停止し、切断される場合があります。
  その場合 [Streaming] 表示が消え、通常の定期取得に切り替わります。
  定期取得時に再度 User Streams へ接続可能になった場合、自動的に再接続されます。
- [表示] > [リアルタイム更新] のメニュー項目のチェックを切り替えることで、
  一時的にリアルタイム更新をしないことができます。(切断はされません)

* クイック検索ボックスについて
- 普通に文字を入力するとその下にある一覧から検索します。インクリメンタル検索が無効な場合は Enter で検索を開始します。
- 先頭が : だと、クエリになります。
- 先頭が / だと、正規表現による検索になります。
- 先頭が @ だと、ユーザ名による検索になります。
- なお先頭が : / @ のいずれかのとき、インクリメンタル検索はその間無効になります。

* Q & A
: 設定画面ないんですか |
  ありません。詳細な環境設定ダイアログは作る予定はありません。断じてありません。今後もありません。
  しかし、実行時に編集できた方が便利な一部の設定 (タブのレイアウトや、フッタなど) は編集ダイアログを用意してあることがあります。
  また、派生品などがもしできる場合はもちろんこの限りではありません。
  (単に面倒くさがりです。)

: タスクトレイに仕舞えませんか |
  <InterfaceSettings> MinimizeToTray または CloseToTray を True にすることで可能です。
  MinimizeToTray は最小化時、CloseToTray は閉じるボタンを押したときに代わりにタスクトレイに格納します。
  良くわからない場合は、Solar を一旦切って Solar.slconf をテキストエディタで開いて MinimizeToTray="False" を検索して MinimizeToTray="True" にして保存して Solar を起動してみましょう。

: WHERE やクエリってなんですか |
  長くなるので次のセクションへどうぞ。

: マルチカラムできますか |
  レイアウトダイアログで (カラム) を選択して左下の [+] を押すとカラムが増やせます。
  ツリー項目のドラッグアンドドロップでタブは移動できますのでどうぞご活用ください。
  ただし、各カラムのサイズは等分割固定です。また、カラムは空にできません。

: URL 短縮できますか |
  投稿ボックスの右クリックメニューにあると思いますので、押すと投稿ボックスにある URL が短縮されます。
  <PostSettings> AutoShortenUrl を活用することで投稿時に自動的に短縮できます。たとえば AutoShortenUrl="TinyURL" とすると TinyURL.py で投稿時に自動短縮します。

: フッタはどうつけるんですか |
  ツールメニューのフッタから設定した後投稿ボックスの右クリックメニューのフッタの項目からチェックできます。
  チェックした項目は、投稿ボックスが空の時に投稿ボックスにフォーカスされると自動的に入力されカーソルが先頭に移動します。
  自動的に入力されたフッタはそのまま投稿ボックスを離れれば投稿ボックスは空になります。
  また、投稿時に自動付加するものではないので、フッタを消して投稿した場合、投稿されたつぶやきにフッタは付きません。

: キャッシュ件数を増やしたいです |
  タブから見れる件数 ≒ キャッシュ なので <TimelineSettings> MaxStatuses を増やすと増えます。

: メモリ食いすぎます |
  タブか、<TimelineSettings> MaxStatuses を減らしましょう。
  またキャッシュを扱うスクリプトを使用している場合、それを減らすのもありかもしれません。

: 複数のアカウントに同時投稿したいです |
  <InterfaceSettings> AccountBoxStyle を CheckBox にしましょう。
  アカウント選択がチェックボックス方式になり、送信先を複数選択したりできます。

: よくサブアカ誤爆します |
  <PostSettings> AutoSwitchAccount="True" のとき、返信や RT 時などはそれを受信したアカウントに自動的に投稿先が切り替わります。
  返信した後アカウントを戻し忘れるのであれば <InterfaceSettings> AccountBoxStyle="CheckBox" にすると間違えづらくなるかもしれません。

: スクロールしたりすると真っ白になります |
  読み込み中の場合は、読み込みを待ってください。
  そうでなければ、<TimelineSettings> ShowIconShadowIfAvailable を False にしてみてください。

: ソース公開しませんか |
  ソースきちゃないので迷っています。

* クエリ
- WHERE やクエリなどと表記されているところの入力は、条件式を入力することで DynamicQuery によりその条件に合うつぶやきを抽出することができます。
- it というキーワードで後述の Status オブジェクトを取得でき、そこからメンバにアクセスできます。
  例: it.IsDirectMessage
  また、it を使用せずそのままメンバ名を書いてもいいです。
  例: IsDirectMessage
- 下位の .NET メソッドも呼び出せます。
  例: Text.Contains("大丈夫だ、問題ない")
- 「キャッシュへクエリ」はローカルで処理するので、いろいろな式を試す場合は便利かもしれません。
- 使用可能な論理演算子は、&& (and), || (or), ! (not) があります。
- 使用可能な比較演算子は、= (==), != (<>), >, <, >=, <= があります。
- わからなければスルーしていいです。

** Status オブジェクト
class Status
{
    bool		IsDirectMessage;		// DM かどうか
    bool		IsSearchResult;			// 検索結果かどうか
    bool		IsRetweet;				// 公式 RT かどうか
    bool		IsOwned;				// 現在のアカウントによるつぶやきかどうか
    bool		IsReply;				// 返信かどうか
    bool		Protected;				// protected かどうか
    bool		Favorited;				// お気に入りされているかどうか
    DateTime	CreatedAt;				// 日時
    long		StatusID;				// つぶやきの ID
    long		InReplyToStatusID;		// 返信先 ID
    string		InReplyToUserName;		// 返信先ユーザ ID 名
    string		Text;					// 本文
    string		UserName;				// ユーザ ID 名
    string		FullUserName;			// ユーザの名前
    string		SourceName;				// ソース名 (送信クライアント名)
    Uri			SourceUri;				// ソース URL (送信クライアント URL)
    Status		RetweetedStatus;		// 公式 RT の場合、RT 元のつぶやき
    Status		RetweetedStatusOrSelf;	// 公式 RT の場合、RT 元のつぶやき。それ以外の場合、このつぶやき自身。
    User		User;					// ユーザ
}

** User オブジェクト
class User
{
    Uri		ProfileImage;		// アイコン
    string	Name;				// ユーザ ID
    string	FullName;			// フルネーム
    string	Location;			// 場所
    string	Description;		// 自己紹介
    Uri		WebSite;			// Web サイト
    bool	Protected;			// protected かどうか
    int		StatusesCount;		// つぶやき数
    int		FollowersCount;		// フォロワー数
    int		FollowingCount;		// フォロー数
    int		FavouritesCount;	// お気に入り数
}

** 例
: 非公式 RT を抽出 |
  not IsRetweet and Text.Contains("RT ")
: 返信なつぶやきを抽出 |
  IsReply
: そんな装備で大丈夫か？ |
  UserName.Contains("enoch")

* 削除
- 設定は Solar.slconf に保存されます。
- レジストリなどの類は使ってないので
  削除するときはまるごと削除しただければ OK です。

* スクリプト
Solar はスクリプトを追加して動作を拡張できるかもしれません。
スクリプトは基本的に Scripts/ にあります。またツールメニューのコンソールも使用できます。
新しく書きたい場合などは Scripts/Help/readme.txt を読みましょう。
